# @repo/db — Postgres + Drizzle data layer (ADR 0032)

**Built package** (consumers resolve `dist/` — build before use; ESM NodeNext,
relative imports need `.js`). Owns the schema, the migration chain, the column
conventions, and the typed factories. Drizzle is pinned at `1.0.0-rc.x`;
driver is node-postgres `Pool` (transaction-pooler-safe — ADR 0038).

## Public surface (the `exports` map — nothing else is importable)

- `@repo/db` — `Db` type + schema barrel; `./client` — `createDb()` factory
  (small pools, `max: 10` default; read-replica seam via primary/replica URLs).
- `./schema/<module>` — **per-module entry points** (`auth`, `outbox`,
  `audit`, `projects`, …): an api module may import only ITS OWN entry
  (ESLint-enforced). `pnpm gen module` adds new ones (`src/schema/index.ts`).
- `./columns` — `id` (UUIDv7), `timestamps()`, `softDelete()`, and **`pii()`**:
  the wrapper that registers a column in the PII registry driving GDPR
  export/erasure and log redaction. Every PII column MUST use it.
- `./pii` — the registry readers; `./factories` — typed test/seed factories;
  `./migrations-path` — for the migrate runner.

## Rules that bite

- Migrations (in `migrations/`, generated by `pnpm db:generate`) are
  expand/contract, N−1 compatible, `lock_timeout = '5s'`, run as a release-
  phase one-shot (`apps/api` `migrate`) — never at boot (ADR 0038).
- Soft-deleted PII is still PII for erasure. No session GUCs, LISTEN/NOTIFY,
  or prepared-statement reliance anywhere (transaction-pooler safety).
- The I3 stores (`release`, `catalog_version`, `price_table`) carry
  BEFORE-UPDATE **immutability triggers** (ADR 0100): any value change to the
  frozen identity/body columns raises — even from raw SQL. A migration that
  must rewrite one (none anticipated) has to drop/recreate the trigger
  explicitly. Lifecycle metadata (status, effective window, …) stays mutable.

Governing ADRs: `docs/adr/0032-postgres-drizzle-db-package.md`, `0038`, `0100`.
