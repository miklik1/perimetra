# ADR 0123 — Wave B: the buyer /nabidka/[token] rebuild — forking the shared document into a branded landing (the ADR-0089 reversal), honest subtraction, and closing the superseded gap

**Status:** Accepted (2026-07-24 — W8 Phase 2, Wave B). Reverses one deliberate half of [ADR 0089](0089-buyer-public-nabidka-view.md) (with Martin's explicit go), applies the design authority (`design/configurator/frames-quote.jsx`), and follows the wave method of [ADR 0119](0119-orders-surface-reskin.md)–[0122](0122-login-two-factor-legal-profile-reskin.md).

## Context

`/nabidka/[token]` is the PUBLIC, unauthenticated buyer conversion surface — the flagship of the Perimetra soft-release ([[Decision — Mercata surfaces + Perimetra soft-release together]]). [ADR 0089](0089-buyer-public-nabidka-view.md) built the endpoint + the accept/decline lifecycle but deliberately deferred "render-taste (the buyer page chrome — minimal vs branded landing) to Martin's eye", and had the route render the SAME `NabidkaDocumentView` the internal rep-print route uses. The design canvas (`frames-quote.jsx`: FrameDesktop/FrameMobile/FrameAccepted) is now that taste decision: a branded landing. Martin's batch "Go" (2026-07-23) adopted the recommendation to **fork** the buyer view into its own landing — a real reversal of ADR 0089's "one shared document, no second surface", recorded here.

## Decision

**Fork the buyer view into a new branded landing, hold the ADR-0089 trust boundary, subtract every canvas overdraw, and close the superseded gap.**

- **The fork.** A new `NabidkaLandingView` (`apps/web/app/nabidka/[token]/nabidka-landing.tsx`) renders at `/nabidka/[token]`. `NabidkaDocumentView` is **completely untouched** — it remains the component the internal `/quotes/:id/nabidka` print route renders (verified: zero diff under `apps/web/app/quotes/[id]/nabidka/`, its tests still pass).
- **Trust boundary preserved (ADR 0089).** The landing renders ONLY `sharedNabidkaSchema` = `{ document, status, validUntil, superseded }` — the real BOM `lines[]`/`categories[]`, the real `tax` breakdown (reverse-charge aware), the real `validUntil`/`superseded`. It adds NO new authed query and surfaces NO cost/margin/site/stamps. One responsive tree with a single `StickyActionBar` (price + Accept/Decline) at every breakpoint — not two desktop/mobile components (a literal duplicate would put two live copies of the same accessible name in the DOM).
- **Honest subtraction (§11.2).** Ten canvas elements omitted because no backend field backs them: the 3D drawing + RAL swatch; the curated spec list (**replaced by the real BOM `lines[]`**); the "V ceně je" perks; the sales-rep contact card; "Mám dotaz" and "Upravit"; "Stáhnout PDF" (the only PDF surface is behind the session auth wall — a public print route is new scope); the "no advance payment" footnote; and — the single largest overdraw — **FrameAccepted's fulfillment timeline and its fabricated order number** (accept only flips `quote.status`; no order is created or returned, so the accepted state is an honest success `Alert`, never an invented order number or fake progress).
- **Closing the real gap (the inverse — a feature the code failed to render).** `superseded` and the `quote_superseded` 409 were returned by the endpoint but unrendered. The landing now shows a warning `Alert` from the flag; `isSupersededConflict` detects the live-race 409 by `ApiError.code` (it carries no `status` field, so `resolvedStatusFrom` fell through to the generic banner before); and because `superseded` is ORTHOGONAL to `status` (ADR-O1/CAR-158 — supersession never rewrites status), the copper "ready/live" badge is suppressed under supersession (the warning banner is the sole state signal), while every non-actionable state falls back to the shared `QuoteStatusBadge` vocabulary.
- **min-h-screen:** the new landing wrapper owns `min-h-screen` + `bg-field`; the `notFound()` branch keeps the global not-found's own `min-h`.

## Consequences

- i18n: 10 new `quotes.buyer.*` keys (cs primary + en parity, ICU-plural validity days); `buyer.accept` → "Závazně objednat" / "Place binding order".
- api-mocks: a 5th seeded quote (`supersededById` set) + the mock accept/decline handlers now 409 `quote_superseded` before the generic not-open check (mirrors the real service's guard order), which is what lets the new live-race test exercise the real code path.
- No schema/endpoint/backend-behaviour change → integration suite not required. I3/immutability untouched (buyer READ only, no issue/verify/snapshot path touched).
- Gate green: `pnpm turbo check-types lint test build --force` 81/81 (web:test 552, api 400); `pnpm knip` clean after registering `capture-wave-a.mjs` in `knip.json`'s `apps/web` entry array — a latent gap from the Wave A commit (its confirming gate ran before that harness was moved into the repo).
- Adversarial review (3 opus dims incl. a dedicated honesty/trust-boundary dim) found 5 issues: **1 medium** — the tax table lacked an `overflow-x-auto` wrapper (unlike the sibling BOM table), so cs-CZ non-breaking-space money strings pushed the page body to horizontal scroll on a narrow mobile column (**fixed**, wrapped like the BOM table) — plus **3 low** (stale RSC JSDoc, a dangling doc reference, and the copper "live" badge appearing on a superseded offer — all **fixed**). **1 low a11y REJECTED** as out of scope: `Panel.Title`/`Alert.Title` render non-heading elements, yielding an h1→h3 outline skip on a resolved-state banner — this is the app-wide kit convention on every reskinned surface, deferred to a `@repo/ui` change (give `Panel.Title` an optional heading element), not a Wave-B regression.
- Eyes-on cleared ×6 ship-bar widths in light + dark (`apps/web/scripts/verify/capture-nabidka.mjs`, new — public, no auth; the 4 buyer states against real seeded quotes: issued 9002, accepted 9003, declined 9004, superseded 9001). **Issued:** brand lockup + copper "NABÍDKA PŘIPRAVENA" badge + the real BOM + a sticky Accept ("Závazně objednat")/Decline bar + the Daňový doklad — no drawing/perks/rep. **Accepted:** the honest "Nabídku jste přijali. Děkujeme." success `Alert`, a neutral PŘIJATO badge, and **no fabricated order number or timeline**. **Superseded:** the closed-gap warning `Alert` ("Nabídka byla nahrazena") with **no copper "ready" badge and no CTA**. **Declined:** an honest info banner. Both themes resolve.
- **Eyes-on caught a real horizontal-overflow bug the gate + the adversarial review missed** (beyond the review's F4 tax-table wrap): at 390 px the dense BOM/tax tables' min-content blew the whole flex/grid chain past the viewport (body 413 > 390). Root cause: the `overflow-x-auto` wrappers AND their grid-column + `Panel` ancestors all carried the default `min-width: auto`, so nothing could shrink below the table's intrinsic width — the [ADR 0121](0121-projects-customers-surface-reskin.md) scroll-wrapper lesson, generalized to a nested **grid + flex** chain (`min-w-0` on the innermost wrapper alone is insufficient when its ancestors are also flex/grid items). Fixed with `min-w-0` on the grid columns, the three `Panel`s, and the two table wrappers; the dense tables now scroll internally in-card on mobile while the body stays 390. Also hardened `capture-nabidka.mjs` to wait for a landing-specific signal ("Dodavatel") rather than a bare `<main>` — a transient api-pool blip renders `notFound()` as a soft-200, which a `<main>`-only wait would silently screenshot as a false pass. (Dev-env note: the api pool intermittently 404'd ~4/30 shots under the capture's rapid sequential load; the endpoint is stable at low request rate — not a product defect.)
