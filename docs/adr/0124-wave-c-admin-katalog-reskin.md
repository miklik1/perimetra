# ADR 0124 — Wave C: the `/admin` Katalog reskin — honest subtraction to the design-canvas Katalog LOOK

**Status:** Accepted (2026-07-24 — W9 Phase 2, Wave C). Applies the design authority (`design/configurator/frames-catalog.jsx`) to the tenant admin surface, following the wave method of [ADR 0119](0119-orders-surface-reskin.md)–[0123](0123-nabidka-buyer-landing-reskin.md) under the contract-honesty rule.

## Context

`/admin` (`apps/web/app/admin/*`) is the **tenant** admin surface. After [ADR 0062](0062-per-tenant-release-visibility.md) retiered authoring to the vendor (`/platform`), what remains here is org-scoped and narrow: the org's **price tables** (publish + version history) and its **upgrade offers** (opt-in version pins, [ADR 0064](0064-release-version-pin-opt-in-upgrade.md)).

The design canvas (`design/configurator/frames-catalog.jsx`) draws an aspirational, org-editable **model IDE**: a product-**family** list (with `+ Přidat rodinu` and per-family `Aktivní/Skryto`), per-family cenová-pravidla tables (`N pravidel`, `+ Pravidlo`), validation **Limity**, editable **Parametry**, a `V nabídce` visibility toggle, `Publikovat verzi` release publishing, and an `Obnovit` (restore) on every version row. **None of that is backend truth in this org-scoped admin** — families/rules/limits/parameters are vendor-authored release data (not org-editable), there is no visibility column or endpoint, release publishing is vendor-only, and price tables are immutable so there is nothing to "restore".

So Wave C is **pure honest subtraction**: adopt the canvas's _visual grammar_, omit every invented datum, and record the divergence here (§11.2).

## Decision

**Reskin the three existing admin pieces to the canvas LOOK on the kit `Panel`/`Field`/`Input` primitives, subtracting every overdraw. No backend change.**

- **`PriceTableForm`** → four sectioned `<Panel elevation="flat">` blocks — _Nastavení ceníku_ (currency / DPH / effective dates / margin floor / rounding / version), _Komponenty_ (the `ArrayField` of code→sell→cost rows + the catalog-code `<datalist>` + the `priceRulesHint` "bez DPH — dopočítává konfigurátor" subtitle), _Výroba a montáž_ (manufacturing rate/multiplier/installation + the `hasCost` toggle + conditional cost fields), and the _Hromadné vložení (JSON)_ island — with a **copper** `Publikovat` submit. It stays **one flat, org-wide price table**: no per-family / per-kind / note columns are invented. All behaviour is preserved unchanged (react-hook-form, `EnumSelect`, `ArrayField`, the bulk-JSON `applyIsland` + parse-error, the duplicate-code guard, the publish mutation). Scalar fields moved from the render-prop `FieldShell` to the compound `Field`/`Field.Control` (ADR 0111); `EnumSelect` is wrapped **inside** each `Controller.render` so `Field.Control` clones the select directly and its `id`/`aria-*` land on the focusable control (label association preserved).
- **`PriceTablesList`** → a **versionRow timeline** (the canvas `versionRow` LOOK): a copper filled `check` glyph + a `Nejnovější` badge on the highest-version table, a muted `draft` glyph on older ones (robust `Math.max` newest-detection, not list-order-dependent), each row `v{version} · {currency} · {effectiveFrom}`. **No `Obnovit`.** Price tables are **immutable (invariant I3)** — a newer version supersedes the prior one from its effective date, so there is nothing to restore; the canvas's restore action is replaced by an honest immutability note (`priceTablesDescription`).
- **`ProductVersions`** → **family-style rows** off the **real** upgrade offers only (`listUpgrades`): a single **generic** `ProductGlyph` (a `modelId` does not identify a gate family, so faking a per-family icon would be dishonest — one neutral glyph for every row), the mono `modelId` title, a `K dispozici novější verze` info badge, a `v{pinned} → v{latest}` sub-line, and the existing per-row `Povýšit` button (pending-label logic + pin mutation kept). The empty state is the honest `noUpgrades` note in a `Panel`.
- **Section headers** use the display face at a controlled `text-xl`, **not** `DisplayLabel`. `DisplayLabel` is a 96px editorial page-hero primitive (used as `h1` across the reskinned surfaces); the repo's `twMerge@3.6` does not classify `text-display` as a font-size utility, so a `DisplayLabel` sub-header would render at 96px and dwarf `SettingsLayout`'s own 24px page `h1`. The reskin keeps the canvas's actual sub-section vocabulary (Chillax display, semibold, `text-xl`).
- **`min-h-screen` per-branch:** the authed branch is `SettingsLayout` (unchanged); the bare `AuthGuard` fallback `<main>` keeps `min-h-screen` and gains `bg-field` (the Wave A rule).

### Omitted overdraws (the §11.2 divergence list)

`Publikovat verzi` (release publish — vendor-only, lives on `/platform`) · the `V nabídce` visibility toggle (no visibility column/endpoint) · `N pravidel` rule counts (no rule-count data) · the editable `Parametry` tab · `+ Pravidlo` / `+ Přidat rodinu` · the per-family **split** of the price table · `Obnovit` (restore — I3). **Slice B** (the product-family list / `Limity` / `Parametry` / `Verze`-as-changelog) is **deferred** out of this wave: its product scope is unresolved (`design/README.md` §8.6 — rule-kind taxonomy, the `Limity`/`Parametry` duplication, and a new visibility column + endpoint), and building against undefined shapes was refused.

## Consequences

- **i18n:** 6 new `admin` keys (cs primary + en parity): `settingsSection`, `manufacturingSection`, `priceRulesHint`, `priceTablesActive`, `priceTablesDescription`, `upgradeAvailable`. (`componentsSectionHint` is retained but now unreferenced — harmless; i18n objects are not knip-checked.)
- **No schema / endpoint / backend-behaviour change** → the integration suite is not required. I3/immutability untouched (admin READ + the pre-existing immutable price-table publish path only).
- **New file** `apps/web/scripts/verify/capture-wave-c.mjs` (authed-only), registered in `knip.json`'s `apps/web` entry array (the Wave B lesson — an unregistered `scripts/verify/*.mjs` harness reports as an unused file).
- **`price-table-form.test.tsx` unchanged** — `Field.Label`+`Field.Control` preserve every `getByLabelText`; `hasCost` was kept as a labelable native checkbox (a kit `Switch` is a button, not label-associable); the bulk-JSON bare `<textarea>` (the test's only-textarea grab) was kept.
- **Gate:** full `turbo check-types lint build --force` green, `web` unit tests green (`--no-file-parallelism`, flake-safe), `knip` clean.
- **Adversarial review:** 3 opus dimensions (correctness + contract-honesty / kit-composition + a11y + responsive / i18n + test-integrity + capture) returned **zero** confirmed findings; the reskin was independently re-read and eyes-on-verified as a faithful subtraction with behaviour preserved.
- **Eyes-on** cleared ×6 ship-bar widths in light + dark (`capture-wave-c.mjs`): the versionRow timeline (`v2 · CZK · NEJNOVĚJŠÍ` + the immutability note, **no restore**), the honest `ProductVersions` empty-state `Panel`, the four sectioned form `Panel`s + copper `Publikovat`; both themes resolve, no horizontal body scroll at 390. (Dev-env noise, not product defects: the known Centrifugo `:3000`-origin WebSocket 403 on this `:3002` box, and React-Query unmount `AbortError`s fired when the capture closes each page after the screenshot.)
